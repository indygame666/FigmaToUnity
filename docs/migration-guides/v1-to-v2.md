# Руководство по миграции: schema v1 -> v2

Этот шаблон описывает, как безопасно применять будущие изменения схемы.

## Обязательные практики

- Всегда указывать `schemaVersion` в каждом экспортируемом пакете
- Добавлять migration-адаптеры в backend до включения нового поведения экспортера
- Перед rollout повторно прогонять golden-тесты

## Чеклист rollout

1. Добавить адаптер `v1 -> v2`
2. Выполнить smoke-проверку импорта на golden-экранах
3. Выполнить реимпорт golden-экранов и сравнить diff-отчеты
4. Включить v2 сначала только для пилотных экранов
